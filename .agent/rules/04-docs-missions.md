# Rule: Docs Missions

**Phase**: Execution

When writing documentation:

1. **Consistency**:
   - Ensure `README.md` stays the source of truth.
   - Update `docs/ARCHITECTURE.md` if components change.
2. **Format**:
   - Use Markdown.
   - Use Mermaid for diagrams.
3. **Verification**:
   - Read the generated file to ensure formatting is correct.
   - Check links.
