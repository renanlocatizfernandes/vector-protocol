## Core Terms

- Bot: Autonomous trading loop controlled via `/api/trading` endpoints and `backend/modules/bot`.
- Supervisor: Safety/health controller exposed via `/api/system/supervisor`.
- Signal: Candidate trade generated by the signal generator before risk validation.
- Trade/Position: Executed order or open position persisted in PostgreSQL (`Trade` model).
- Risk Manager: Module that enforces exposure, leverage, and stop constraints.
- Sniper: Specialized trade mode for targeted executions (see sniper endpoints in `backend/api/routes/trading.py`).

## Acronyms & Abbreviations

- API: Application Programming Interface (FastAPI backend).
- PnL: Profit and Loss (reported by metrics endpoints).
- SL/TP/TSL: Stop Loss, Take Profit, Trailing Stop Loss.
- DB: PostgreSQL database.
- WS: WebSocket (used for live logs/updates).

## Personas / Actors

- Operator: Monitors health, starts/stops the bot, and inspects logs.
- Trader: Tunes strategies, risk parameters, and execution settings.
- Developer: Extends modules and maintains API/UX surfaces.

## Domain Rules & Invariants

- Trades must pass risk validation before execution.
- Configuration changes should be persisted and reviewed for impact.
- Exchange API errors should halt or pause execution rather than retry aggressively.
